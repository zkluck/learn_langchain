# Stage 09：生产化改造

## 目标

- 从 Demo 升级到可上线方案
- 补齐安全、可观测、可恢复能力

## 改造清单

1. 安全

- 工具调用前增加安全网关
- 高危命令白名单/黑名单
- workspace 越权访问防护

2. 可用性

- 模型回退（多 provider 或多模型）
- 重试 + 熔断 + 超时策略
- MCP server 健康检查

3. 可观测

- 结构化日志（JSON）
- trace_id / session_id / tool_call_id 全链路
- 关键指标：成功率、平均步数、超时率、失败分布

4. 数据层

- Session Note 从本地 JSON 升级到 DB/向量库
- 摘要内容可持久化并可检索

5. 部署

- 容器化 + 非 root 运行
- CPU/内存/磁盘限额
- 环境变量与密钥托管

## 验收标准

- 有一套可执行的上线检查清单
- 关键失败路径都可追踪（日志与指标）
- 在压力场景下可降级而不崩溃
